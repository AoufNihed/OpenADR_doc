Overview

The Smart Energy Management System leverages machine learning and OpenADR protocols to predict energy demand, optimize production, and automate decision-making. This project integrates predictive analytics with demand-response mechanisms to ensure efficient energy management.

Features

  • Energy Demand Prediction: Utilizes Random Forest regression to forecast daily energy demand based on environmental factors.
  • Threshold Setting: Establishes dynamic thresholds for energy production optimization.
  • Demand-Response System: Implements OpenADR protocols for automated production adjustments.
  • Interactive Dashboard: Provides real-time visualization of predictions and system decisions.

OpenADR Implementation

VTN (Virtual Top Node)

The VTN is responsible for sending signals to VENs based on predicted energy demand.

class VTN:
    def __init__(self):
        self.signals = []

    def send_signal(self, ven_id, action, adjustment):
        signal = {"ven_id": ven_id, "action": action, "adjustment": adjustment}
        self.signals.append(signal)
        print(f"Signal sent to VEN {ven_id}: {action} with {adjustment}")
        return signal

VEN (Virtual End Node)

The VEN receives and processes signals from the VTN to adjust energy production.

class VEN:
    def __init__(self, ven_id):
        self.ven_id = ven_id

    def receive_signal(self, signal):
        if signal["ven_id"] == self.ven_id:
            print(f"VEN {self.ven_id} received signal: {signal['action']} - {signal['adjustment']}")
        else:
            print(f"Signal not intended for VEN {self.ven_id}")

Example Workflow

vtn = VTN()
ven = VEN(ven_id="client_1")

# Simulate a prediction and decision
predicted_demand = 1050 
threshold = 1000

if predicted_demand > threshold:
    action = "Raise Production"
    adjustment = f"Increase production by {predicted_demand - threshold:.2f} MW"
else:
    action = "Optimize Production"
    adjustment = f"Reduce production by {threshold - predicted_demand:.2f} MW"

# VTN sends signal to VEN
signal = vtn.send_signal(ven.ven_id, action, adjustment)

# VEN processes the signal
ven.receive_signal(signal)

API Documentation

Endpoints

  • GET /predict - Returns the predicted energy demand.
  • POST /adjust - Sends a signal to adjust energy production.

Example Requests

curl -X GET http://localhost:5000/predict
curl -X POST http://localhost:5000/predict -d '{"action": "Raise Production", "adjustment": "50 MW"}'
